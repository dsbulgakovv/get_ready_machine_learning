# Databases Handbook

Этот модуль закрывает базовые вопросы по БД, которые часто всплывают у ML Engineer: типы хранилищ, SQL vs NoSQL, транзакционность и практические различия между системами.

## Основы

### Какие бывают типы баз данных?

Основные группы:

- реляционные;
- документные;
- key-value;
- column-family;
- графовые;
- time-series;
- in-memory.

### В чём разница между реляционными и нереляционными базами данных?

- `Реляционные`: таблицы, схема, SQL, связи, транзакции.
- `Нереляционные`: более гибкая модель данных, часто проще масштабируются под конкретный тип нагрузки.

### Почему в некоторых задачах предпочтительнее реляционные базы?

Потому что они дают:

- строгую схему;
- JOIN'ы;
- ACID-транзакции;
- богатый SQL;
- удобство аналитики и контроля целостности данных.

### Что такое транзакционность?

Это выполнение группы операций как единого атомарного блока. В контексте ACID обычно имеют в виду:

- Atomicity;
- Consistency;
- Isolation;
- Durability.

### Что такое SQL?

Это язык запросов к реляционным БД: выборка, фильтрация, агрегация, обновление, описание схемы.

### Приведи пример реляционной базы данных

PostgreSQL, MySQL, SQLite, MS SQL Server.

### Приведи пример нереляционной базы данных

MongoDB, Redis, Cassandra, Neo4j.

### В чём особенность MongoDB?

Это документная NoSQL БД. Удобна, когда данные естественно хранятся как JSON-подобные документы и схема может быть гибкой. Но за гибкость иногда платят сложностью аналитики и контроля связности.

### С какими базами данных ты работал?

На интервью лучше отвечать честно и по существу: назвать системы и тип нагрузки. Например: PostgreSQL для транзакционных и аналитических задач, Redis для кэша, ClickHouse для событийной аналитики, MongoDB для документного хранения.

## Что важно запомнить по модулю

- SQL vs NoSQL — это не "что лучше", а "что подходит под модель данных и нагрузку".
- Для MLE обычно особенно важны PostgreSQL, Redis, объектные хранилища и иногда векторные БД.
- Если в ответе упоминаешь БД, полезно сразу добавлять: зачем использовали именно её.
